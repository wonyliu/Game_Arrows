# 主界面设计知识库 V1 - 参考拆解

## 1. 参考样本与定位
- `R1`：强主视觉 + 单主按钮首页，视觉情绪清晰，适合开场页。
- `R2`：系统化大厅，入口密度高但层级稳定，适合功能扩展页。
- `R3`：轻量活动页，按钮语义直接，移动端触控区友好。

V1 采用“混合路线”：首页借鉴 `R1/R3` 的轻卡通主 CTA 语言，功能区借鉴 `R2` 的入口组织方式，避免当前界面“元素多但没有主次”的问题。

## 2. 布局骨架提炼（430x932 基准）
- 纵向三段：
  - 顶部信息区（约 19%-22%）：品牌、返回、资源信息。
  - 中部主操作区（约 33%-38%）：主标题、主 CTA、当前关卡标签。
  - 底部功能区（约 24%-30%）：功能卡片网格与角标。
- 保留稳定外框比例，不让任一模块“随屏撑满”导致上下区不可点击。
- 控件布局优先级：`主 CTA > 功能卡 > 辅助说明 > 装饰元素`。

## 3. 视觉层级语法
- 色彩语义：
  - 大背景使用天空/草地浅色区分氛围层。
  - 可交互面板统一暖木色/奶油色，保证信息可读。
  - 强交互按钮使用高对比暖黄绿，不与背景同明度。
- 轮廓语义：
  - 面板有明确描边和阴影，不和背景融为一体。
  - 主按钮采用“凸起”视觉，次级按钮使用低饱和或描边态。
- 信息语义：
  - 主标题一屏只保留一个焦点词组。
  - 数值信息（金币/体力）统一胶囊样式，避免散落。

## 4. 按钮与入口语法
- 主按钮（Primary CTA）：
  - 一屏仅一个。
  - 宽度显著大于其他按钮，文本长度短且动词明确。
- 功能入口（Feature Entry）：
  - 同层最多 6 个。
  - 统一尺寸、统一图标尺度、统一文案长度策略。
  - 角标只用于“待处理/奖励/新增”，不做装饰。
- 返回与关闭：
  - 语义固定（左上返回、右上关闭或面板头返回），避免多处重复。

## 5. 信息密度与留白规则
- 单屏可见强交互元素建议上限：8-10 个。
- 横向最小间距：8px；纵向主区间距：12-16px。
- 安全留白：
  - 顶部 >= 24px（适配刘海区）
  - 底部 >= 24px（适配手势条）
  - 左右 >= 16px

## 6. 当前版本常见混乱反例（用于避免）
- 一个页面同时出现多个“看起来像主按钮”的操作。
- 角标过多，造成视觉噪声。
- 装饰图纹进入按钮文字区，导致可读性下降。
- 同级入口尺寸不一致，导致用户误判优先级。
- 文案长度无约束，英文切换后挤压布局。

## 7. 落地原则
- 参考图只学习结构与视觉语法，不复用任何第三方原图元素。
- 先形成“规则库 + 配置库”，再驱动 UI 重绘和 Gemini 出图。
