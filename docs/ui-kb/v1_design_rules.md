# 主界面设计知识库 V1 - 可执行规则

## 1. 目标
- 把“界面太乱”拆成可校验规则，供设计、前端实现和 AI 出图共用。
- 适配当前菜单状态机：`HOME / SETTINGS / LEADERBOARD / SKINS / CHECKIN / EXIT_CONFIRM`。

## 2. 硬规则（MUST）
| 规则ID | 规则 | 阈值 |
|---|---|---|
| R-CTA-1 | 每屏主 CTA 数量 | `= 1` |
| R-ENTRY-1 | 同层功能入口数量上限 | `<= 6` |
| R-BADGE-1 | 每屏同时可见角标数量上限 | `<= 2` |
| R-TAP-1 | 所有可点击控件最小触控区 | `>= 44 x 44` |
| R-SAFE-1 | 顶部安全边距 | `>= 24` |
| R-SAFE-2 | 底部安全边距 | `>= 24` |
| R-SAFE-3 | 左右安全边距 | `>= 16` |
| R-LANG-1 | 主按钮文案长度（zh-CN） | `<= 6 字` |
| R-LANG-2 | 主按钮文案长度（en-US） | `<= 18 chars` |
| R-LANG-3 | 功能入口文案长度（zh-CN） | `<= 4 字` |
| R-LANG-4 | 功能入口文案长度（en-US） | `<= 12 chars` |
| R-NOISE-1 | 按钮文字区域纹理对比 | 不可抢文本层级 |

## 3. 视觉与层级规则
- 层级优先：`主 CTA > 功能入口 > 次要操作 > 装饰`。
- 色彩语义：
  - 背景层为低对比浅色氛围。
  - 交互层统一暖色系面板基底，文本保持高对比。
  - 禁止多种高饱和色同时竞争主视线。
- 描边与阴影：
  - 主按钮、功能卡、面板必须有明确外轮廓。
  - 同层组件阴影强度一致，不做随机变化。

## 4. 组件规则
- 主按钮（`primary_cta_button`）：
  - 仅用于主流程动作（开始游戏、确认退出）。
  - 不承载角标、不承载复杂图标。
- 功能卡（`feature_card`）：
  - 图标上半区、文案下半区。
  - 默认显示 5 个入口：设置、排行榜、皮肤、签到、退出。
  - 禁用态降低饱和度和对比度，不改变尺寸。
- 资源胶囊（`resource_capsule`）：
  - 只显示数值类信息，不放跳转动作。
- 角标（`badge_indicator`）：
  - 仅用于 `checkin/leaderboard` 等待处理信息。
  - 数字优先，超过 99 显示 `99+`。
- 面板头（`panel_header`）：
  - 左侧返回语义固定，标题居中。
  - 一屏仅一个面板头。

## 5. 本地化规则
- 以 `zh-CN` 为主设计基线，`en-US` 必须可容纳不换行或最多两行。
- 所有文案走 i18n key，不允许模板硬编码业务文案。
- 英文文案超长时优先缩短语义，不允许缩放字体到不可读。

## 6. AI 出图规则（用于 Gemini）
- 必须：
  - 可爱卡通蛇洞主题。
  - 低噪点、清晰边缘、按钮文字区纯净。
  - 输出 PNG；按钮/卡片/图标使用透明背景。
- 禁止：
  - 在图标上烘焙文字。
  - 花纹覆盖按钮文本区。
  - 过多装饰按钮（缎带、挂件、复杂描边叠加）。

## 7. 回退与兼容
- 主题回退链固定：`design-v5 -> design-v3 -> design-v2`。
- 知识库只定义规则与映射，不替代现有 manifest 资源加载机制。
