## 2. 明早你需要完成的人工步骤（账号登录相关）

下面步骤必须你本人登录平台完成，我无法代替。

### 2.1 微信小游戏（必做）

1. 登录微信公众平台（小游戏主体管理员账号）。
2. 确认已开通“流量主”。
3. 新建激励视频广告位，至少 1 个：
   - 用于 `support_author`
4. 复制广告位 `adUnitId`。
5. 打开游戏后台 `admin.html` -> 参数管理：
   - 填入 `supportAds.adUnitIds.support_author`
   - 确认 `support_author` 开启，其他位按你策略保持关闭/开启。
6. 保存参数并在前台实测一次“支持作者”按钮。

### 2.2 Google Play / AdMob（必做）

1. 登录 [AdMob](https://admob.google.com/)。
2. 创建 Android App（若未创建），新建激励广告位。
3. 记录：
   - App ID
   - Rewarded Ad Unit ID（用于 `support_author`）
4. 登录 [Google Play Console](https://play.google.com/console)：
   - 创建应用条目
   - 补齐商店信息、隐私政策、内容分级、数据安全表单。
5. 准备 Android 包（AAB）并上传到内部测试轨道。
6. 内测安装后验证“支持作者”激励广告可以完成并回调奖励。

## 3. 重要可行性说明（双平台）

- 当前仓库是 H5 + Node 后端架构，微信小游戏与 Google Play 都需要各自的“平台壳/导出链路”。
- 这次已把广告、配额、后台配置、玩家持久化链路打通，属于“商业化核心逻辑落地”。
- 平台最终提审前，你至少还需要：
  - 微信：小游戏工程打包并通过开发者工具真机调试。
  - Android：AAB 构建与签名、Play Console 提审资料完备。

## 4. 生产安全与上线门槛（必须）

生产环境请启用：

- `ADMIN_REQUIRE_KEY=1`
- `ADMIN_API_KEY=<强随机密钥>`
- `USER_CENTER_BACKEND=postgres`
- `USER_CENTER_DATABASE_URL=<生产库连接串>`

并确认：

- 后台接口不裸奔公网
- DB 有备份策略
- 日志里无明文密钥

## 5. 我建议的上线节奏（明天执行）

1. 先发小流量（或内测包）验证广告闭环与玩家留存。
2. 保持仅 `support_author` 激励位 24 小时观察数据。
3. 若数据稳定，再决定是否开启第 2 个激励位（`fail_continue` 或 `double_coin` 二选一）。
4. 每日根据数据调 `defaultDailyLimit` 与玩家级 `dailyLimitOverride`。
