# 美术资源调用判定架构

## 判定目标

美术资源管理的“使用/未使用”标准不是“代码里是否出现过路径”，而是：

- 玩家当前版本运行时可能看见的图片资源，判定为“已使用”。
- 只存在于概念稿、制作中间产物、后台工具、已删除 UI 元素、未启用 CSS 变量里的图片，判定为“未使用”或“制作产物”。

当前策略版本：`player-visible-runtime-v2`。

## 已计入的运行时引用源

- `index.html`。
- `js/` 下的玩家运行时代码。
- `css/` 下的玩家运行时样式。
- 已完成且可玩的皮肤目录，包括完整散件皮肤和 `atlas-sheet` 皮肤。

## 明确排除的引用源

- `js/admin-*` 后台管理脚本。
- 后台编辑器、制作工具、皮肤适配工具里的引用。
- `assets/**/concepts/**` 概念稿路径。
- 文件名带 `_raw`、`_source`、`_export` 的制作中间文件。
- 已在 `data/managed-config/ui-layout-config-v1.json` 中删除的 UI 元素对应资源。

## CSS 变量规则

CSS 中仅声明变量不等于玩家可见。例如：

```css
--asset-home-snake-down: url('../assets/ui/home/components/snake_pose_down.png');
```

只有该变量被实际 `var(--asset-home-snake-down)` 使用，才进入候选引用。若对应 UI 元素已经被界面编辑器删除，即使 CSS 仍保留备用声明，也不计为玩家可见。

## UI 元素归属规则

部分 UI 元素通过 CSS 变量挂载图片。扫描器维护“元素 -> 资源路径”的归属表：

- `home.homeBgSnakeUp -> assets/ui/home/components/snake_pose_up.png`
- `home.homeBgSnakeDown -> assets/ui/home/components/snake_pose_down.png`

如果该元素出现在 `deletedElements` 中，对应图片不再算玩家可见。

## 皮肤资源规则

皮肤不依赖普通全文路径扫描，而是单独按运行时皮肤规则计入：

- `assets/skins/<skinId>/manifest.json` 中 `mechanism: "atlas-sheet"` 的皮肤，使用 `manifest.assets` 中声明的运行时资源。
- 非 atlas 皮肤必须具备完整的蛇头、表情、身体、尾巴散件，才判定为可玩皮肤。
- `assets/skins/templates/**` 只作为模板，不属于玩家可见皮肤。

## 自检接口

开发服务器提供：

```text
GET /api/admin/assets/self-check
```

该接口会跑固定回归用例，验证典型误判不会反复出现，例如：

- 旧道具图标 `icon_shuffle.png`、`icon_undo.png`、`icon_hint.png` 不应算已使用。
- 已删除的 `snake_pose_down.png` 不应算已使用。
- `settings_panel_a.png` 是概念稿，不应算已使用。
- 当前主界面按钮和背景图必须算已使用。

## 已知边界

静态扫描不能 100% 推断任意动态拼接路径。新增可见 UI 图片时，应满足其中一种方式：

- 在运行时代码或实际使用的 CSS 中出现明确路径。
- 若是可编辑 UI 元素图片，补充到 `PLAYER_VISIBLE_UI_ASSET_OWNERS`。
- 若是皮肤资源，写入标准皮肤目录或 `manifest.assets`。

删除资源前仍建议先看“判定原因”和预览，避免动态拼接路径未登记造成漏判。
